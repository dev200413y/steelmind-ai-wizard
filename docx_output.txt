2. Background Theory — Agentic AI Explained from Scratch
This section explains all theoretical concepts from basics so that anyone — even without prior AI knowledge — can understand the system.
2.1 What is an AI Agent?
Simple Definition
An AI Agent is a software system that can PERCEIVE its environment, REASON about what to do, TAKE ACTIONS, and LEARN from results — all with minimal human intervention.
Think of it like this: A simple chatbot just answers questions. An AI Agent actually does things — it can search the web, run code, call APIs, make decisions, and even hire other agents to help it.

2.2 What is Multi-Agent AI?
Instead of one big AI doing everything, Multi-Agent AI divides the work among SPECIALIZED agents — just like a hospital has different doctors for different specialties, not one doctor for everything.
SINGLE AGENT (old way):
  User query → One LLM → One answer
  Problem: One model cannot be expert in everything
MULTI-AGENT (modern way):
  User query → Orchestrator → assigns to specialists
  Vision Expert + Database Expert + Reasoning Expert + Report Expert
  Each agent does what it is best at → combined answer is far superior
2.3 What is RAG — Retrieval Augmented Generation?
RAG in Simple Words
RAG means: before the AI answers your question, it first SEARCHES through your own documents (manuals, SOPs, logs) and retrieves the most relevant information. Then it uses that information to give a grounded, accurate answer — not just a guess.
WITHOUT RAG:
  Q: What is the maintenance procedure for Blast Furnace tuyere?
  A: (AI guesses from training data — may be wrong or outdated)
WITH RAG:
  Q: What is the maintenance procedure for Blast Furnace tuyere?
  Step 1: Search Tata Steel Blast Furnace Manual (your own document)
  Step 2: Find relevant 3-4 paragraphs about tuyere maintenance
  Step 3: AI reads those paragraphs + answers with full citation
  A: (Accurate, grounded in your actual SOP, traceable to page number)
RAG is what makes the AI trustworthy in industrial settings. Every answer has a source — engineer can verify it.
2.4 What is LangGraph?
LangGraph is a framework for building multi-agent AI systems as a graph — where each node is an agent and edges define how information flows between agents. It supports:
Parallel execution — multiple agents run simultaneously
Conditional routing — orchestrator decides which agents to call based on input type
State management — shared memory across all agents in the graph
Human-in-the-loop — pause and ask human for approval at any point

2.6 What is STT and TTS?
STT (Speech-to-Text): Converts spoken audio into written text. Used when engineer speaks a query — Whisper model transcribes it to text which the AI can process.
TTS (Text-to-Speech): Converts written text into spoken audio. Used to speak the AI response back to the engineer — so they can hear the answer hands-free while working.
2.7 What is Anomaly Detection?
Anomaly detection means automatically finding data points that are unusual or abnormal. In a steel plant, if vibration sensor suddenly reads 3x its normal value, that is an anomaly. The Isolation Forest algorithm is used — it learns what 'normal' looks like from historical data and flags anything that deviates significantly.
2.8 What is RUL — Remaining Useful Life?
RUL is a prediction of how many more days/hours a piece of equipment can safely operate before it fails or needs replacement. It is calculated by analyzing sensor degradation trends over time. Early RUL warning allows planned maintenance instead of emergency breakdown.

7.1 Orchestrator Agent — The Brain
Role
The Orchestrator is the central controller. It receives all inputs, decides which specialist agents to call, manages the shared state, and assembles the final response. It is NOT a chatbot — it is a router and coordinator.
Property
Details
Input
All user inputs (voice text, image path, CSV path, PDF path, query text)
Output
Agent execution plan — which agents to call, in what order/parallel
LLM
Mistral 7B Instruct via Groq API
Framework
LangGraph StateGraph
Key Logic
Conditional routing based on input type — image? call Vision. CSV? call Anomaly.
# Orchestrator routing logic (pseudocode)
def route(state):
    agents = []
    if state['has_image']:   agents.append('vision_agent')
    if state['has_csv']:     agents.append('anomaly_agent')
    if state['has_docs'] or state['text_query']:
                             agents.append('rag_agent')
    return agents  # these run IN PARALLEL

8. True Agentic AI — Parallel + Autonomous Agents
Critical Point for Judges
Most submissions will be sequential pipelines (Input → Agent1 → Agent2 → Output). That is NOT Agentic AI — that is just a chain. True Agentic AI means agents run in PARALLEL, make AUTONOMOUS decisions about what to do next, and COMMUNICATE with each other based on intermediate results.
8.1 Parallel Execution
When a user submits an image + CSV + text query, the Orchestrator dispatches Vision Agent, Anomaly Agent, and RAG Agent SIMULTANEOUSLY — not one after another. All three run in parallel threads and return their results to the shared state. Diagnostic Agent then synthesizes all results.
# LangGraph parallel execution
graph.add_node('orchestrator', orchestrate)
graph.add_node('vision_agent', analyze_image)
graph.add_node('rag_agent', retrieve_knowledge)
graph.add_node('anomaly_agent', detect_anomalies)
graph.add_node('diagnostic_agent', diagnose)
# Fan-out: orchestrator → [vision, rag, anomaly] in PARALLEL
graph.add_conditional_edges('orchestrator', route_parallel,
    {'vision': 'vision_agent', 'rag': 'rag_agent', 'anomaly': 'anomaly_agent'})
# Fan-in: all parallel agents → diagnostic
graph.add_edge('vision_agent', 'diagnostic_agent')
graph.add_edge('rag_agent', 'diagnostic_agent')
graph.add_edge('anomaly_agent', 'diagnostic_agent')

8.3 Inter-Agent Communication
Agents react to each other's outputs. This is what makes it truly agentic:
If Anomaly Agent flags CRITICAL temperature spike → Diagnostic Agent automatically escalates urgency
If Vision Agent detects crack in pressure vessel → Risk Scorer automatically assigns CRITICAL regardless of other factors
If RAG Agent finds a similar historical failure with known fix → Diagnostic Agent prioritizes that solution
If Risk Scorer outputs CRITICAL → Alert Agent fires immediate notification without waiting for full report

13. Anomaly Detection — Sensor Analysis
13.1 How Isolation Forest Works
Isolation Forest is an unsupervised machine learning algorithm that detects anomalies by trying to 'isolate' data points. Normal points require many splits to isolate (they cluster together). Anomalous points are isolated quickly with few splits. The anomaly score is based on how quickly a point was isolated.
Normal reading:  Temperature = 1200°C  → needs many splits to isolate → NORMAL
Anomalous:       Temperature = 1680°C  → isolated in 2 splits → ANOMALY (score 0.9)
Threshold: anomaly_score > 0.6 → flag as anomaly
           anomaly_score > 0.8 → trigger CRITICAL alert
13.2 RUL Prediction Method
Remaining Useful Life is predicted using linear regression on rolling average degradation trend. If vibration has been increasing 0.3 mm/s per day and safe limit is 4.5 mm/s, current reading is 3.6 mm/s → RUL = (4.5 - 3.6) / 0.3 = 3 days. Trigger maintenance alert with 7-day buffer.
